## P1：高价值架构改进

### 增加部署后线上 URL 可访问性检查

**现状**

- Pages artifact 已包含 `web/`、`art/` 和 `public/`，并有本地测试保护。
- 当前仍缺少自动访问线上 URL 并检查主要图片资源 HTTP 状态的步骤。

**建议后续重构**

- 增加部署后 smoke check，检查 `/web/`、`/art/shop.png`、`/public/assets/ui/cards/deck_back.png` 等关键路径。
- 将检查结果写入发布记录或 CI 摘要。

### 拆分 `web/app.js` 的职责

**现状**

`app.js` 仍同时负责：

- 输入事件。
- 商店。
- 红眼生命周期提交。
- 结算流程。
- 领域动画调度。

**风险**

- 修改局部流程容易影响无关区域。
- 难以为完整回合编写测试。

**建议后续重构**

- 继续提取商店、红眼生命周期和领域动画编排。
- 保留现有行为和数据结构，避免一次性重写。

### 统一玩家术语与内部术语

**现状**

规则核心仍大量使用 `pressure`、`module`、`redline`、`ignitionSequence` 等历史命名。

**风险**

- 新开发者容易误判当前产品概念。
- 文档与代码检索成本高。

**建议后续重构**

- 在稳定测试保护下逐步统一内部命名。
- 不要只做全局替换，应先明确数据契约。

### 为动画建立统一调度边界

**现状**

- 动画依赖多个 `wait`、定时器、class 和动态克隆节点。
- 资源动画与 CSS 动画分别触发。

**风险**

- 动画中断、快速重开或阶段切换时可能残留节点/class。
- 动画时长调整容易破坏流程时序。

**建议后续重构**

- 建立可取消的动画调度器。
- 统一清理临时 DOM、class 和定时器。
- 保留资源加载失败时的降级路径。
