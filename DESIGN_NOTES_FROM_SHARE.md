# 深渊引擎设计要点

来源：ChatGPT 分享对话 `https://chatgpt.com/share/6a055e4c-2c30-83ec-822a-f20ed52ed869`

本文只沉淀对项目有执行价值的设计判断，不复述完整对话。

## 产品定位

《深渊引擎》不应该只是 Balatro 换皮，而应该是：

> 带压力成本的工业扑克牌 + 红温风险投资引擎。

卡牌构筑只是表达语言，真正差异化来自“玩家主动追求危险”的情绪体验。

## 市场判断

Steam 独游的机会不在“又一个卡牌肉鸽”，而在机制嫁接：

- Deckbuilder 已经变成底层语言，可以嫁接生存、自动化、模拟、社交传播和情绪体验。
- 玩家购买的不只是策略，而是 build 幻想、重复游玩、系统自发生成的故事。
- 独游爆款要有一句话 Hook、低理解成本、高重复性、可传播瞬间。
- 纯内容堆叠型项目风险高，系统生成型项目更适合小团队。

对本项目的结论：

- 保留卡牌构筑的清晰输入。
- 把差异化集中在压力、红温、提现、机器反馈和 Joker 规则污染。
- 不要扩成“卡牌 + 生存 + 模拟 + 自动化 + 剧情”的混合怪。

## 复杂度分层

必须严格分层，否则学习成本会爆炸。

### 基础牌

基础牌是输入材料，不是技能牌。

只允许三要素：

- 相位：`⚙ / ⚡ / 🔥 / ☣`
- 筹码：`10 / 20 / 30 / 40 / 50`
- 压力：`▲1 / ▲3 / ▲5 / ▲8 / ▲12`

基础牌职责：

- 提供基础功率。
- 构成牌型。
- 增加压力成本。

基础牌禁止：

- 条件触发。
- 随机效果。
- 红区特殊效果。
- 降压、防爆、重复触发。
- 大段说明和工业术语。

### 牌型

牌型负责让基础牌产生结构意义。

当前改为直接复用 Balatro 的标准扑克牌型：

- 高牌、对子、两对、三条。
- 顺子、同花、葫芦、四条、同花顺。

牌型只提供固定基础分和倍率，不引入额外资源，也不写触发文本。

### Joker

Joker 是规则修改器，负责让朴素基础牌变疯。

红温流第一套核心：

- 压力越高，倍率越高。
- 进入红区后，倍率翻倍。
- 压力超过 90，基础功率翻倍。
- 每次进入红区，永久倍率 +1。
- 进入红区后，最后一张牌重复触发。

## 情绪设计

项目的核心验证不是“牌能不能算分”，而是：

> 玩家会不会主动把机器推向红区。

需要制造三种情绪：

- 贪婪：明明已经能过，但红区收益更香。
- 压迫：机器正在呻吟，继续拉杆有生理压力。
- 后悔：爆炸不是随机惩罚，而是自己贪出来的结果。

## 可借鉴的情绪结构

这些不是玩法抄袭对象，而是情绪参考。

- Frostpunk：机器过载必须有声音、震动、发红、失控感。压力必须是物理存在，不只是数字。
- Luck Be a Landlord：系统要逐步出现自动连锁，让玩家觉得机器自己活起来，而不是只在算数学题。
- Loop Hero：加入“继续榨 / 立刻提现”的风险选择，让失败可归因。
- Quacks of Quedlinburg：爆炸不一定直接 Game Over，可以是长线残疾惩罚，避免玩家过度保守。
- Cultist Simulator：后期可引入桌面压力、驻留卡、倒计时威胁，但不进第一版 MVP。
- Buckshot Roulette / FTL / Inscryption：分别参考死亡悬停感、残破抢修感、规则突破感，作为后续方向。

## 当前禁令

在 MVP 验证前，不做以下内容：

- 不扩 20 种流派。
- 不堆长文本世界观。
- 不继续加复杂基础牌。
- 不做大量特殊牌。
- 不做完整商店经济。
- 不做 Boss 和章节结构。

当前只验证：

> 红温是否让玩家上头。
