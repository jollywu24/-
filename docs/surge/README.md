# 暗涌

## 目录说明

本目录目前只用于说明暗涌领域。当前没有独立的 `surge` 代码目录。

## 当前定义

暗涌牌是使用红眼赌注后，从真实牌堆额外抽取的一张牌。

它：

- 进入弃牌堆。
- 不进入中央出牌区。
- 不参与牌型。
- 不提供普通单牌筹码。
- 牌面点数会增加上头值。
- 可以触发部分赌鬼效果。

## 当前实现位置

### 纯规则

`web/logic-pure.js`：

- `cardHypeValue`：将牌面转换为暗涌上头值，`J/Q/K = 10`，`A = 11`。
- `redEyeHypePreview`：无具体暗涌牌时返回 `2~11` 风险范围。
- `calculateHypeBreakdown`：将暗涌上头值计入结算。
- `applyRedEyeBet`：处理与暗涌相关的赌鬼倍率。
- `simulatePreview`：返回 `surgeCard`、`hypeFromSurgeCard` 和最终上头值。

### 浏览器流程与演出

`web/app.js`：

- `showdown`：有红眼赌注时，在调用 `simulatePreview` 前从牌堆抽一张暗涌牌并立即放入弃牌堆。
- `animateSurgeReveal`：从右上牌堆飞出、翻面，并将上头值反馈飞向表盘。

`web/style.css`：

- `.surge-reveal-card`
- `.surge-card-face`
- `.surge-card-back`
- `.surge-hype-fly`
- 暗涌相关关键帧

## 与赌鬼的关系

- `红眼借据`：暗涌翻牌后额外增加 `3` 上头值。
- `小牌壮胆`：暗涌点数不高于 `5` 时，本手倍率增加该点数。

## 已知问题

- 暗涌抽牌发生在 `app.js`，暗涌规则发生在 `logic-pure.js`，演出也在 `app.js`，理解流程需要跨文件。
- 当前没有独立的暗涌结果数据结构。
- 动画与上头值更新时序通过固定等待时间串联。
- 暗涌牌在规则计算前已经从真实牌堆移入弃牌堆，流程中断时缺少统一回滚边界。

## 建议后续重构

- 定义明确的暗涌结果对象：牌、点数、额外修正、最终上头值变化。
- 将抽牌、规则计算和动画播放分成三个稳定步骤。
- 增加“牌堆为空并从弃牌堆补充”情况下的流程测试。
