# 决策记录

## 当前产品方向

项目现在定义为《深渊赌局》：一款黑暗扑克肉鸽，核心围绕赌博风险、赌鬼修正和继续贪下去的诱惑。

当前核心钩子：

> 玩家明知道再贪一手可能会毁掉赌局，但还是想按下 `摊牌`。

在核心赌博循环被验证之前，项目不应漂移成大型工业世界、重剧情 roguelite 或地图冒险。

## 主要差异化

项目不只和 Balatro 拼组合数学爽感。

目标情绪生态位是：

- 风险管理
- 贪婪
- 崩盘边缘
- 自己作死后的后悔
- 危险收益追逐

当前 UI 通过以下语言表达：

- `上头值`
- `红眼`
- `鬼压桌`
- `红眼赌注`
- 黑暗赌桌表现

## 固定画布决策

主 UI 是固定游戏桌：

- 基准尺寸 `1600 x 900`
- 整体等比缩放
- 不做单个面板独立响应式重排

原因：这个界面是被构图过的游戏桌，不是网页。

## 主界面范围决策

主界面只保留五个常驻区域：

- 左侧信息栏
- 顶部赌鬼区
- 中央出牌区
- 底部手牌区
- 右侧操作区

常驻商店、日志、无事件支撑的额外弹窗和重复分数面板都不放在主游玩界面。

## 术语决策

当前视觉方向使用赌博术语：

- `赌鬼`，玩家可见 UI 不叫 Joker。
- `上头值`，玩家可见 UI 不叫压力。
- `摊牌`，不叫拉杆或 Pull。
- `换牌`，不叫刷新。
- `红眼赌注`，不叫过载模块。

历史代码或内部逻辑中可以保留旧工业术语，但除非有意重新引入，否则不要主导玩家可见 UI。

## 扑克基础决策

基础牌使用标准扑克基础：

- 52 张牌
- 4 个花色
- 13 个点数，从 2 到 A
- 抽牌堆和弃牌堆

原因：扑克牌型提供直觉数学锚点。玩家无需学习自定义符号系统，就能理解对子、两对、顺子、同花和葫芦。

## 手牌可读性决策

底部手牌默认自动排序：

- 点数从大到小。
- 同点数按 `♠ > ♥ > ♦ > ♣`。
- 排序发生在发牌、补牌、换牌补入和进入下一轮重新发牌后。
- 选择、hover 和结算过程中不重新排序。

原因：底部手牌是玩家识别对子、三条、顺子和同花的主要区域。排序必须提升可读性，但不能改变抽牌堆、弃牌堆或 seed 结果。

## 牌动画稳定性决策

发牌和补牌动画使用独立飞牌克隆承载移动与翻面，真实手牌在目标位置冻结并隐藏，直到克隆牌完全对齐后再无过渡切换显示。

原因：如果真实手牌和克隆牌同时参与 opacity、transform 或选中态过渡，动画末尾会出现闪烁、明暗变化或最后几张牌位置跳动。飞牌动画的最后一帧必须优先稳定坐标和亮度，而不是追求额外的缓动效果。

## 复杂度分层决策

复杂度分层如下：

1. 基础牌是简单输入。
2. 扑克牌型提供基础筹码和倍率。
3. 赌鬼/Joker 修正规则。
4. 商店、卡包、牌型升级和 Boss 盲注提供长期决策。
5. 红眼系统提供短期危险诱惑。

基础牌必须保持简单，这样破坏规则的卡才会显得特殊。

## 红眼与上头值决策

关键验证目标是玩家是否会主动追逐危险。

`上头值` 不能只是装饰性仪表。它要影响收益、风险和玩家情绪。理想瞬间应该像：

- “我可以安全过，但这手更贪。”
- “我知道这可能要崩。”
- “这把输是我自己的问题。”

跨轮规则：

- `上头值` 是跨轮风险资源。
- 最大 `上头值` 为 `160`。
- `上头值 >= 100` 进入红眼，降至 `80` 或以下退出红眼。
- 红眼状态下，本手倍率 `×1.5`。
- 普通关、精英关、Boss 关达成目标后分别减少 `25`、`35`、`50` 上头值。
- `红眼赌注` 选择不是跨轮增益，进入下一轮时重置。
- 红眼赌注的风险来自暗涌牌：它用一张未知暗涌牌的点数替代本手正常上头增长，让玩家在“确定风险”和“浮动风险”之间下注。
- 下一轮开始时，`红眼赌注` 入口重新锁定，玩家必须再次进入危险状态才会获得新选择。

结果优先级：

1. `上头值 >= 160` 立即爆牌，当前赌局失败。
2. 如果没有爆牌且分数达到目标，进入通关结算；确认后进入下一轮。
3. 如果摊牌次数耗尽且分数仍低于目标，庄家通吃，当前赌局失败。

## 商店可见性决策

商店存在于盲注之间，不作为主界面常驻区域。

原因：主界面必须专注于手牌、分数、上头值、牌堆和操作按钮。商店决策重要，但不应和核心牌桌抢注意力。

## 当前 MVP 状态记录原则

本文件只记录设计方向和原因，不维护最新功能清单。

最新玩法规则以 `docs/component_rules.md` 为准；实现位置和改动导航以 `PROJECT_CONTEXT.md` 与 `ARCHITECTURE.md` 为准；待办优先级以 `TASKS.md` 为准。

## 下一步设计优先级

最高优先级仍然是验证情绪循环：

1. 玩家能快速理解当前手牌。
2. 玩家能看到安全选择和贪婪选择。
3. 玩家会主动选择危险。
4. 玩家过度贪婪后会后悔，但想立刻再开一局。
