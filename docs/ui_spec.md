# UI 规格

## 画布

- 基准画布：`1600 x 900`。
- 整个游戏画布居中，并根据浏览器视口等比缩放。
- 页面不滚动。
- 这是固定游戏 UI 画布，不是响应式网页布局。单个面板不应独立重排。

## 主界面布局

主游玩界面只有五个常驻区域：

1. 左侧信息栏。
2. 顶部赌鬼区。
3. 中央出牌区。
4. 底部手牌区。
5. 右侧操作区。

不要在主游玩界面增加常驻商店、日志、弹窗、地图或额外侧栏。

允许的非常驻状态层只有：

- `红眼赌注` 选择弹窗。
- 通关后的 `结算收益` 弹窗。
- 失败后的 `爆牌` / `庄家通吃` 弹窗。

这些状态层只在对应事件触发时出现，关闭或进入下一局后必须回到五区域主界面。

## 左侧信息栏

用途：按固定顺序展示所有关键局内信息。

必须保持以下顺序：

1. 游戏标题：`深渊赌局`。
2. 按钮：`比赛信息`。
3. 目标分数：`目标分数`。
4. 当前分数：`当前分数`。
5. 当前牌型：`当前牌型`。
6. 牌型等级：`等级`。
7. 筹码 × 倍率：`筹码 × 倍率`。
8. 赌资：`赌资`。
9. 上头值：`上头值`。
10. 上头值阶段：`冷手`、`热手`、`红眼`、`鬼压桌`。
11. 摊牌次数：`摊牌`。
12. 换牌次数：`换牌`。

左侧信息栏顺序受保护。视觉打磨时不要重排。

## 顶部赌鬼区

- 标题：`赌鬼`。
- 只显示已经获得的赌鬼卡。
- 初始化时没有自带赌鬼牌，计数为 `0 / 5`。
- 赌鬼卡代表长期规则修正。
- 静态状态下应可见，但暗于中央出牌区。
- 只有触发时才应该变亮。

## 中央出牌区

- 只显示已经通过 `摊牌` 提交的牌。
- 已出牌比手牌更端正，略微更亮。
- 不在这里显示牌型名、公式、重复分数或预览数学。
- 未摊牌前，选中的手牌不能出现在这里。

## 底部手牌区

- 显示 8 张当前手牌，扇形排列。
- 选中牌轻微上浮，并使用橙金色高亮。
- 手牌计数显示 `8 / 8`。
- 扇形必须留在中央游玩区域内，不能压到右侧操作区。

## 右侧操作区

必须保持以下顺序：

1. 牌堆。
2. 剩余牌数。
3. `摊牌` 按钮。
4. `换牌` 按钮。
5. 锁定的 `红眼赌注` 入口。

`红眼赌注` 默认锁定。它应显得危险但未激活，视觉权重不能超过 `摊牌` 按钮。

## 状态弹窗

### 红眼赌注弹窗

- 只在上头值达到红眼阈值、右侧入口解锁并点击入口后出现。
- 每次出现 3 个候选项。
- 视觉上使用暗红细边框、顶部红眼装饰和旧赌注卡样式。
- 选择后弹窗关闭，右侧入口只保留小图标和状态文字。
- 详情只通过 tooltip 展示，不在右侧展开成长面板。

### 红眼全局状态表现

- 画布通过 `normal`、`preheat`、`red-eye`、`critical`、`betting`、`bust` class 表达当前风险状态。
- 红眼状态可以影响表盘、牌桌边缘、手牌边缘、红倍率和暗涌牌堆，但不能改变五区布局。
- 红眼赌注结算优先表现暗涌牌飞出、翻面、点数揭示和上头值飞向表盘。
- 动画必须短促，不阻塞核心操作。

### 通关结算弹窗

- 只在当前分数达到目标或被红眼赌注判定为达标后出现。
- 展示 `本轮收益`、`奖励明细`、`总计收益` 和 `进入下一局` 按钮。
- 奖励数字可以逐行播放，但不能新增常驻结算栏。
- 点击 `进入下一局` 后，收益进入左侧 `赌资`，弹窗消失并发下一局手牌。

### 失败弹窗

- `上头值 >= 160` 时显示 `爆牌`。
- 摊牌次数耗尽且未达标时显示 `庄家通吃`。
- 失败弹窗可以展示失败原因和关键数值，但不改变左侧信息栏顺序。
- `再来一局` 重置当前演示局。

## 主界面保护规则

- 不在主游玩界面添加常驻商店 UI。
- 不常驻显示红眼赌注选项。
- 红眼赌注选项只在触发时作为中央弹窗出现。
- 选择红眼赌注后，只显示小状态图标，详情通过 tooltip 展示。
- 除非明确要求，不重新引入工业拉杆或老虎机界面。
